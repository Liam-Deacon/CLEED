1 /****************************************************************************
2  PATTERN (Georg Held 22.03.93)
3 NAME
4  pattern -- LEED pattern visualisation program
5 
6 DESCRIPTION
7  Pattern simulates Low Energy Electron Diffraction (LEED) patterns
8 to help in analysis of complex surface structures. Diffraction peaks
9 (spots), indices, reciprocal lattice vectors & different domains can
10 all be visualised using vector graphics (PostScript).
11 
12 INPUT FILE
13  The input file specifies the coordinate system, substrate (GS) and
14 superstructure (SS), as well as any SS domains. The file has the following
15 format:
16  * lines beginning with '#' are ignored by the program.
17  * lines beginning with 'c' are treated as comments, note that
18  the first 'c' comment block will be printed as a legend.
19  * real space lattice vectors a1 & a2 have the form: <x> <y> a[1-2]
20  * radius is the maximum radius of Ewald construction in units of: a[1-2]
21  * number of domains
22  * superstructure matrix M1 & M2 of the form: <x> <y> M[1-2]
23  * domain mirroring or rotation: Sx, Sy or R<#>
24 
25 BUGS
26  Please report any bugs (with a description on how to reproduce
27 the problem) to liam.m.deacon@gmail.com
