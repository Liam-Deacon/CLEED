28 /************************************************************************
29 
30  Transformation of the multiple scattering matrix from angular momentum
31  space into k-space.
32 
33  Mk'k = pre_fac * sum(l,m,l',m') |k'><l,m|M(lm,l'm')|l'm'><k| + dkk'
34 
35  Ylm = |k'><l,m| = Ylm(k') (k_max rows) x ((l_max+1)^2 columns)
36  Yxlm = |k><l',m'| = Y*l'm'(k) ((l_max+1)^2 rows) x (k_max columns)
37 
38  pre_fac = i*8*PI^2 / (|k|*A*kz+)
39 
40  INPUT:
41 
42  mat Mkk - output: multiple scattering matrix in k-space.
43  mat Mlm - input: multiple scattering matrix in (l,m)-space.
44 
45  mat Ylm - input: transformation matrix for exit beams (l,m) -> (k) of
46  dimension: n_beams x (l_max+1)^2 (e.g. output from leed_ms_ymat)
47  mat Yxlm - input: transformation matrix for incoming beams (k) -> (l,m)
48  of dimension (l_max+1)^2 x n_beams (e.g. output from ms_ytoyx)
49 
50  If either Ykl or Yxkl are NULL, it will be calculated as the
51  conjugate transposed matrix from the opposite transformation
52  using ms_ytoyx.
53 
54  leed_beam_t * beams - additional information about the k vectors
55  involvedi (in this case: (A kz)^-1).
56  The order of beams must be equal to the first dimension of
57  Ykl (not checked).
58 
59  real unsc - relative intensity of the unscattered beams (either 1. for
60  transmission or 0. for reflection)
61 
62  real rel_area - area of the unit cell relative to 1x1.
63 
64 
65  DESIGN
66 
67  Depending on the k-vectors involved in the transformation matrices Ylm
68  and Yxlm, either a reflection (+,- or -,+) or a transmission matrix
69  (+,+ or -,-) will be calculated.
70 
71  RETURN VALUES:
72 
73  NULL if failed (and EXIT_ON_ERROR is not defined)
74 
75  Mkk (may be different from input parameter). The storage scheme for
76  Mkk is defined by the order of Ylm and Yxlm.
77 
78 *************************************************************************/
79 {

153 #endif /* FIND_Yx */

172  Loop over k' (exit beams: rows of Mk'k):

198  *(Mkk->rel+i_r) += unsc;
199  }
200  return(Mkk);
